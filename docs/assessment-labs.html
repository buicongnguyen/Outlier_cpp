Important scope

These are original, assessment-style exercises based on competencies described in public Outlier role pages. They are not copied from, endorsed by, or guaranteed to match any private assessment. The timer, 165-minute mock, and 75-point target are study devices. Outlier's public pages do not disclose an exact C++ format, timer, syllabus, allowed-tools policy, or pass mark; project requirements can vary.

How to answer like a strong code expert

1

Verdict

State the result first: correct, incorrect, or best-to-worst ranking.

2

Evidence

Name the violated invariant and give a concrete failing interleaving or input.

3

Repair

Provide the smallest correct patch, with ownership and error behavior explicit.

4

Tests

Cover normal behavior, boundaries, adversarial cases, and the reported regression.

5

Tradeoffs

Explain complexity, lifetime, synchronization, maintainability, and assumptions.

Full mock: 165 minutes

  1. First pass: read all prompts, mark constraints, and do not open hints.
  2. Execution: work in task order; stop when the master timer reaches zero.
  3. Review: reveal solutions only after submission or expiry.
  4. Scoring: award points only where your draft contains objective evidence.

Recommended weekly loop

  • Day 1: one debugging task, then reproduce it under a sanitizer.
  • Day 2: one coding task with warnings-as-errors and edge tests.
  • Day 3: one response-review or rubric task, emphasizing concise writing.
  • Day 4: repeat the weakest task without hints.
  • Day 5: take the full mock and record the causes of every lost point.

Local C++ check

Task blocks focus on the assessed logic and omit repetitive #include lines. Add the standard headers named by the types you use; the common set below compiles all reference snippets when combined with the relevant task declarations and a test main.

GCC or Clang

g++ -std=c++20 -Wall -Wextra -Werror -pedantic -g task.cpp && ./a.out
g++ -std=c++20 -g -O1 -fsanitize=address,undefined task.cpp && ./a.out
g++ -std=c++20 -g -O1 -fsanitize=thread task.cpp && ./a.out

MSVC Developer Prompt

cl /std:c++20 /permissive- /EHsc /W4 /WX task.cpp

Use a separate TSan-capable Clang/GCC environment for race detection.
